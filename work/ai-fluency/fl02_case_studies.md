## Case Study: MediCare Pro Extra

**The problem**
I saw a real gap - small pharmacies often run on paper logs or 
spreadsheets, with no easy way to track stock levels, catch expiring 
medicine before it's too late, or keep sales records straight. I 
wanted to build something that actually solved that, not just a 
tutorial project.

**What I did (and what I decided)**
I built a pharmacy management system in Python from the ground up. I 
designed the database myself in SQLite - 8 tables and 15 indexes, 
tuned with PRAGMAs for data integrity. I added SHA-256 authenticated 
login with parameterized queries, so it wasn't just functional but 
actually secure. The hardest part wasn't any single piece - it was 
getting everything to work together end-to-end: login, database 
writes, and a background alert system for low stock and expiring 
medicine, all running without breaking each other. Once it worked, I 
packaged the whole thing into a standalone Windows executable with 
PyInstaller, so it runs without anyone needing to install Python.

**What came of it**
A working, installable pharmacy management application - not a demo, 
a real .exe someone could open and use. The full source, schema, and 
structure are public on GitHub for anyone to inspect.
